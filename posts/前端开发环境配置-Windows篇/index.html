前端开发环境配置-Windows篇

2021.09.02

许久未使用 Windows，配置前端开发所需的 npm 和 yarn 时，遇到了各种错误，解决完毕后梳理成文，记录在此。

node、npm 安装及配置

首先，打开 nodejs 官网 https://nodejs.org/en/download/，下载安装软件 .msi 格式。 然后安装一路 Next 下去即可，安装目录自行选择。安装完成后可使用命令查看版本信息。

node -v
npm -v

然后设置 node 安装软件的全局安装目录和缓存目录

npm config set prefix "D:\develop\nodejs\node_global"
npm config set cache "D:\develop\nodejs\node_cache"
npm config set strict-sll false

接着设置环境变量

  • 系统环境变量 NODE_PATH 为 D:\develop\nodejs\node_global\node_modules
  • 用户环境变量 Path 中的 npm 路径 C:\Users\xxx\AppData\Local\Yarn\Data\global 为 D:\develop\nodejs\node_global
  • 在系统环境变量 Path 中新增 D:\develop\nodejs\node_global\bin 路径

yarn 安装及配置

安装好上述的 npm 后, 使用以下命令安装 yarn

npm install yarn -g

安装完成后，可查看 yarn 版本

yarn -v

接着我们需要同步配置 yarn 安装软件的全局安装目录和缓存目录

yarn config set global-folder "D:\\develop\\yarn\\global"
yarn config set cache-folder "D:\\develop\\yarn\\node"
yarn config set strict-ssl false

如果本地网络受限，需要针对 npm 和 yarn 设置代理，使用以下命令

npm config set no-proxy 不走代理的域名
npm config set proxy 代理url
npm config set https-proxy 代理url

yarn config set no-proxy 不走代理的域名
yarn config set proxy 代理url
yarn config set https-proxy 代理url

至此，前端开发环境所需要的 node, npm, yarn 配置完成。

node npm yarn
